## 7. Custom Cursor Scoping (`/admin` and `/properties` routes)
- **Action**: Modified the CSS custom cursor hiding rules to be scoped under a dynamic `.custom-cursor-active` class.
- **Why**: The default browser cursor was hidden globally via `cursor: none !important` in `globals.css` on desktop viewports. However, the custom trailing luxury cursor is only loaded in `(main)/layout.tsx` and is not loaded in layouts outside `(main)` like the admin portal (`/admin`) or properties index (`/properties`). This left these portals with no visible cursor.
- **Fix**: Added code to `components/animations/CustomCursor.tsx` to add `custom-cursor-active` to `document.documentElement` when mounted and clean it up on unmount. Replaced the global cursor hiding rule in `globals.css` with a selector targeting `.custom-cursor-active`, restoring standard browser cursors on `/admin` and `/properties` pages.
